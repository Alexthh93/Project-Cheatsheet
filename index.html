Semantic Rules

Semantic rules have a lot to say, even if someone believes it isn't of importance, someone not taking advantages of correct semantic setup, would at the long run, be in for a lot of extra working hours.
Semantic rules applied correct, wont just help making both HTML, as CSS code more readable, but also makes the working process more available and easy for anyone who would come across, to
either work on, or continuing the current HTML or CSS files. Another inportant ruleset to understand, would be how SEO (Search Engine Optimization) works. When applied correct semantic setup, helps SEO value of ones website, in short Semantic SEO Helps You Earn Featured Snippets.

Important Semantic options

  • <header> as first
  • <article> inside of <section>
  • <img> inside of <figure>
  • <section> instead of <div>
  • <footer> instead of <div>

By applying these unique identifiers we boost the search engines changes of priotizing our page. Since <div> isn't specific enough compare to and <article> when someone searches after blog,
or anything related with information. Using <footer> helps when someone searches for address, since <footer> has more value and is more specific, to address, names, nr etc than pure <div>tag.
