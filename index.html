Semantic Rules

Semantic rules have a lot to say, even if someone believes it isn't of importance, someone not taking advantages of correct semantic setup, would at the long run, be in for a lot of extra working hours.
Semantic rules applied correct, wont just help making both HTML, as CSS code more readable, but also makes the working process more available and easy for anyone who would come across, to
either work on, or continuing the current HTML or CSS files. Another inportant ruleset to understand, would be how SEO (Search Engine Optimization) works. When applied correct semantic setup, helps SEO value of ones website, in short Semantic SEO Helps You Earn Featured Snippets.

Identifier Important Semantic rule
<header> -- at start and as intro SEO values header elements, higher than HTML DIV tags
<article> -- inside of <section> SEO values articles higher than divs, also helps readabillity
<img> -- inside of <figure> Figure tag, is more easy for SEO to locate and value
<section> -- instead of <div> Section, is more specific and easier to change for futre adjustments
<footer> -- instead of <div> SEO optimization, and also values footers for more credibillity

By applying these unique identifiers we boost the search engines chances of priotizing our page since <div> isn't specific enough compared to an <article> when someone searches after blog,
or anything related with information. Using <footer> helps when someone searches for address, since <footer> has more value and is more specific, to address, names, nr etc than pure <div> tag.

CSS

CSS, also known as Cascading stylesheet, is the colors of the homepage. When we think about a website we need to understand that pure HTML, simply sets the skelet of the website, and CSS brings colors and life. By manipulating the selectors, classes or ID's of the HTML files, colors, fonts, borders and other creative designed properties can be added, it all depends on ones imagination.

Classes is most of the type another extension of an HTML element type, we use classes on mostly "articles, sections and footers" etc. That way we can manipualte a big section of the page due to the parents of nested elements on our HTML site. Should we choose to go in further details and manipulate childrens of a class type, we add either another class and makes a more described specification of a given design, which we want to bring to life, or we simply give that unique element an ID class with # hashtag sign.
